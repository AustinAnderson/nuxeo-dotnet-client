1 # Nuxeo .NET Client #
2 
3 The Nuxeo .NET Client is a cross-platform client library developed in C# for the Nuxeo Automation and REST API.
4 
5 Nuxeo .NET Client targets two platforms: `net45` and `dnxcore50`. This allows it to be used to develop apps for not only Windows but also Linux and Mac OS. Bellow follows an app/platform compatibility table for the Nuxeo .NET Client library:
6 
7 ```
8 +------------------------------------+------------+
9 | | Compatible |
10 +------------------------------------+------------+
11 | Destkop .NET 4.5+ | Yes |
12 | ASP .NET 4.5+ | Yes |
13 | ASP .NET 5 (Native & CoreCLR) | Yes |
14 | DNX Console App (Native & CoreCLR) | Yes |
15 | Windows 8/8.1 Store APP | No |
16 | Windows Phone 8/8.1 | No |
17 | Universal Windows Platform (UWP) | No |
18 | Xamarin.Android | Yes? |
19 | Xamarin.iOS | Yes? |
20 +------------------------------------+------------+
21 ```

128 ## Running the Tests ##

150 ## Known Issues ##

152 There is currently a bug where DELETE requests are sent as GET on Linux and Mac OS only, which prevents operations such as deleting documents and drop batches from working. This issue was reported [here](https://github.com/dotnet/corefx/issues/4134). **This bug causes 5 out of 9 tests of the TCK to fail**, on the last stage where all temporary files are deleted.

155 ## Tutorial ##
156 
157 Let's make a quick DNX Console App that creates a folder on the server's root and a child document. This quit tutorial assumes that you have an instance of Nuxeo Server running on your local machine.
158 
159 * Create a new directory for your project named *TestNuxeoApp* and `cd` there
160 
161 * Create a new *project.json* with dependencies to "System.Console" and "NuxeoClient". It should look something like this:
162 
163 ```json
164 {
165  "version": "1.0.0-*",
166  "description": "Nuxeo Test App",
167  "authors": [
168  "John Doe"
169  ],
170  "tags": [
171  ""
172  ],
173  "projectUrl": "",
174  "licenseUrl": "",
175  "dependencies": {
176  "System.Console": "4.0.0-beta-23409",
177  "NuxeoClient": "1.0.0-*"
178  },
179  "frameworks": {
180  "dnxcore50": {}
181  }
182 }
183 ```
184 
185 * Inside the project directory, create a new file named *Program.cs*. Make it create a new Nuxeo Client, create a folder in the server's root directory named "My Folder", and then a file inside My Folder named "My File". It should look something like this:
186 
187 ```csharp
188 using System;
189 using System.Threading.Tasks;
190 using NuxeoClient;
191 using NuxeoClient.Wrappers;
192 
193 namespace TestNuxeoApp
194 {
195  public class Program
196  {
197  public static void Main(string[] args)
198  {
199  // create a new Nuxeo Client, which is a disposable objects
200  using(var client = new Client())
201  {
202  // let's create an async job that doesn't block
203  Task.Run(async () => {
204  // perform an async request to create a folder on the root with the title "My Folder"
205  Document folder = (Document)await client.Operation("Document.Create")
206  .SetInput("doc:/")
207  .SetParameter("type", "Folder")
208  .SetParameter("name", "MyFolder")
209  .SetParameter("properties", new ParamProperties { { "dc:title", "My Folder" } })
210  .Execute();
211 
212  // print the returned folder object if it is not null
213  Console.WriteLine(folder == null ? "Failed to create folder." : "Created " + folder.Path);
214 
215  // perform an async request to create a child file named "My File"
216  Document file = (Document)await folder.Post(new Document {
217  Type = "File",
218  Name = "MyFile",
219  Properties = new Properties { { "dc:title", "My File" } }
220  });
221 
222  // print the returned file object if it is not null
223  Console.WriteLine(file == null ? "Failed to create file." : "Created " + file.Path);
224  }).Wait(); // let's wait for the task to complete
225  }
226  }
227  }
228 }
229 ```
230 
231 * Restore dependencies by running `dnu restore`
232 
233 * Run the app by running `dnx run`
234 
235 * Check the server and verify that the documents were created
